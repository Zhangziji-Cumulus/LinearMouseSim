# 三个已知BUG修复方案

## BUG 1：快捷键失效，需要改为P键

### 根因分析

当前快捷键系统使用 `WH_KEYBOARD_LL` 低级键盘钩子（`hotkey_manager.py:343`）。在全屏游戏中，游戏通常使用 DirectInput / Raw Input 绕过 Windows 消息队列，导致钩子无法捕获按键。代码中已有 `GetAsyncKeyState` 轮询回退机制（`_fallback_to_polling`，第360行），但仅在钩子安装失败时启用。

此外，当前配置文件 `config.json:3` 中 toggle 快捷键为 `]`，需要改为 `p`。

### 修复方案

**文件：`config/config_manager.py`**
- 第7行：将默认值 `']'` 改为 `'p'`

**文件：`config.json`**
- 第3行：将 `"toggle": "]"` 改为 `"toggle": "p"`

**文件：`hotkey/hotkey_manager.py`**
- 修改 `load_hotkeys()` 方法（第302行）：在安装钩子后，同时启动轮询线程作为主要检测方式（而非仅作为回退）
- 轮询间隔从20ms调整为8ms（`_polling_loop` 第394行），确保游戏内按键响应及时
- 这样无论钩子是否生效，轮询都能检测到P键按下

## BUG 2：开启模拟后鼠标未隐藏

### 根因分析

`mouse_capture.py` 的 `hide_cursor_force()`（第80行）通过 `SetSystemCursor` 将系统光标替换为空图标来隐藏。`state_machine.py:82` 在 `turn_on()` 中调用它。

问题在于：`hide_cursor_force()` 中第85行有早返回判断 `if _empty_cursor: return`，但 `_empty_cursor` 在 `restore_cursor()` 中被 `DestroyCursor` 并置 `None`（第129-130行）。所以第二次开启时会重新创建。这部分逻辑正确。

真正的问题可能是：空光标的创建方式（全零位图）在某些系统上可能显示为默认光标而非透明。需要验证空光标是否真的不可见。

### 修复方案

**文件：`core/mouse_capture.py`**
- 重写 `hide_cursor_force()`：改用 `CreateCursor` + 全零 AND/XOR 掩码创建真正的透明光标，确保在所有系统上都不可见
- 移除 `_empty_cursor` 的缓存判断，每次调用都确保光标被正确隐藏

**文件：`core/state_machine.py`**
- 第58-59行：在光标锁定循环中，确保每帧都调用 `SetCursor(_empty_cursor)` 而非仅在 `_empty_cursor` 存在时调用

## BUG 3：游戏中操作与程序界面手感不一致

### 根因分析

核心问题在 `main.py:255-256`：

```python
normalized_angle = max(-1.0, min(1.0, angle / self.config.get('steering.max_angle', 90)))
self.vjoy.set_axes(left_x=normalized_angle, right_trigger=gas, left_trigger=brake)
```

`set_axes()` 方法（`vgamepad_output.py:84`）发送所有轴后调用 `update()`。但问题在于：

1. **油门/刹车是二值的（0或1）**，而真实赛车游戏需要模拟量渐变。右键按下 = 100%油门，松开 = 0%，没有渐变过程。

2. **`set_axes()` 每次都重置所有轴**，包括 `right_joystick_float(x=0, y=0)`，这会覆盖之前可能设置的值。

3. **vgamepad 的 `update()` 调用频率**：主循环 5ms 一帧（200Hz），每次调用 `update()` 发送完整状态到 ViGEmBus 驱动。这个频率足够，但需要确保角度值确实被正确传递。

4. **死区未应用**：`vgamepad_output.py:29` 定义了 `self.deadzone = 0.02`，但 `set_axes()` 不使用它。小角度输入可能被驱动层的死区过滤掉。

### 修复方案

**文件：`core/vgamepad_output.py`**
- 在 `set_axes()` 中应用死区处理：小角度直接归零，避免驱动层死区导致的非线性
- 移除对 `right_joystick` 的不必要重置

**文件：`main.py`**
- 第250-256行：增加调试日志，打印实际发送到虚拟手柄的 normalized_angle 值，方便排查
- 油门/刹车改为可配置的渐变模式（可选功能）

**文件：`core/vgamepad_output.py`**
- 添加 `set_steering_only(left_x)` 方法：仅更新方向盘轴，不触碰油门/刹车轴，避免每次 update 重置所有状态

## 验证方法

1. **快捷键**：启动程序，按P键验证模拟开关是否生效；在游戏内按P键验证是否生效
2. **鼠标隐藏**：开启模拟后确认鼠标完全不可见；关闭模拟后鼠标恢复
3. **虚拟手柄输出**：通过程序日志确认 normalized_angle 值正确；在游戏中验证方向盘响应
