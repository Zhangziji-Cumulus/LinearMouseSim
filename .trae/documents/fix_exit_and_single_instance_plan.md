# 修复程序退出和单实例问题的实现计划

## 问题分析

### 1. 程序关闭后鼠标锁定未释放
**问题根源**：
- `main.py` 的 `cleanup()` 方法虽然调用了 `release_cursor_safety()`，但 `atexit.register` 在程序异常终止时可能不执行
- `SteeringStateMachine.stop()` 调用 `turn_off()`，但 `turn_off()` 只恢复鼠标位置，未调用 `ClipCursorManager.unlock()`
- `ClipCursorManager.lock_to_center()` 被调用后，`unlock()` 可能未被正确触发

### 2. 单实例运行未实现
**问题根源**：
- 当前代码没有检查是否已有实例在运行
- 用户可以多次启动程序导致冲突

### 3. 关闭按钮最小化到托盘
**问题根源**：
- `MainWindow._on_close()` 调用 `self.hide_to_tray()`，而不是真正关闭程序

### 4. 托盘模式下快捷键问题
**问题根源**：
- `keyboard` 库需要在后台持续运行才能捕获全局热键
- 需要确保托盘模式下热键监听线程不被暂停

## 修改方案

### 文件修改清单

| 文件 | 修改内容 |
|------|----------|
| `main.py` | 添加单实例检查、增强退出清理逻辑、添加信号处理 |
| `ui/main_window.py` | 修改关闭按钮行为为真正退出、添加退出确认对话框 |
| `core/state_machine.py` | 确保光标锁定在停止时正确释放 |
| `ui/tray_manager.py` | 确保退出时调用完整清理流程 |

### 修改步骤

#### 步骤 1：修改 `main.py`

**目标**：添加单实例检查、增强退出清理、添加信号处理

**修改内容**：
1. 添加 `mutex` 单实例检查（使用 Windows Mutex）
2. 添加 `signal` 信号处理（捕获 SIGINT/SIGTERM）
3. 在 `__init__` 中注册信号处理器
4. 确保 `cleanup()` 在所有退出路径中被调用

#### 步骤 2：修改 `ui/main_window.py`

**目标**：关闭按钮真正退出程序

**修改内容**：
1. 修改 `_on_close()` 方法，添加退出确认对话框
2. 确认后调用 `self.app.cleanup()` 和 `self.quit()`

#### 步骤 3：修改 `core/state_machine.py`

**目标**：确保光标锁定正确释放

**修改内容**：
1. 在 `turn_off()` 中调用 `self.cursor_manager.unlock()`
2. 在 `stop()` 方法中确保 `_turning_off` 标志设置后立即释放光标

#### 步骤 4：修改 `ui/tray_manager.py`

**目标**：确保退出时调用完整清理流程

**修改内容**：
1. 在 `_on_exit()` 中确保先调用 `self.app.cleanup()`
2. 确保托盘图标正确清理

## 潜在风险和注意事项

1. **单实例检查**：使用 Windows Mutex 需要管理员权限，普通用户可能无法正常工作
2. **光标锁定释放**：需要确保在所有退出路径（正常退出、异常退出、强制关闭）都能释放光标
3. **热键监听**：`keyboard` 库在某些情况下可能需要管理员权限
4. **线程安全**：状态切换需要保持线程安全

## 测试验证

1. **正常关闭测试**：启动程序 → 开启模拟 → 点击关闭按钮 → 确认退出 → 检查鼠标是否解锁
2. **异常关闭测试**：启动程序 → 开启模拟 → 任务管理器强制结束 → 检查鼠标是否解锁
3. **单实例测试**：启动程序 → 再次启动 → 检查是否提示已有实例运行
4. **托盘快捷键测试**：启动程序 → 最小化到托盘 → 使用快捷键 → 检查是否生效
